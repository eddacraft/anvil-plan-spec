## Release Theme

**Team Rollout** — v0.6 finished the monorepo tiers; v0.7 makes APS adoptable
by a *team*, not just its author. The cut targets everything a new team hits
in week one: the fence-parsing false positive that torpedoes first-run trust
(ISS-001), no central lint enforcement (GitHub Action), harnesses without a
prompt entry point (Copilot/Codex/Grok stubs), no machine-readable status
surface for non-CLI stakeholders (`aps export --json`), and no written team
conventions (rollout guide + multi-owner example). Alongside: the D-040
harness-set revision — Gemini out, Grok Build in.

## Risks

| Risk | Impact | Mitigation |
| ---- | ------ | ---------- |
| Fence-awareness changes item indexing for existing plans that (ab)used the bug | Low | Behaviour only changes inside fenced blocks; parity fixtures pin the new contract |
| Grok Build conventions are young and may move | Medium | D-040 binds to the stable `.agents/skills/` + `AGENTS.md` family, not Grok-private paths |
| Export shape becomes a de-facto API before it stabilises | Medium | Top-level `schema` version field; documented as v1-unstable in docs/integrations.md |
| Action pins a release binary that predates export | Low | Action version input defaults to the v0.7.0 tag once cut |

## Out of Scope

- GitHub issue/project sync (exploratory; no work item)
- PowerShell surface for `export` (matches `next`/`rollup` precedent)
- Marketplace publication of the Action (needs a tagged release first;
  dogfood consumption ships now)

## Rollout

1. Land the six work items above on `main` via PR(s), plans kept current.
2. Prep PR: bump 0.6.0 → 0.7.0 across channels; promote CHANGELOG; draft
   `docs/release-notes/v0.7.0.md`.
3. Tag `v0.7.0`; workflow builds five targets + `SHA256SUMS`; crates.io;
   Scoop from published hash.
4. Closeout: this doc → Shipped with evidence; status sweep
   (items `Merged → Released/Shipped → Complete`).
