# Release Plan: v0.6.0

| Field            | Value                                |
| ---------------- | ------------------------------------ |
| Target           | v0.6.0                               |
| Cut from         | `main`                               |

## Release Theme

**The Tagged Tier, For Real** — v0.5 scaled APS to federated plan trees; v0.6
gives the *default* monorepo tier (one canonical index + `Packages:` tags) the
tooling it never had. Tags are now validated, queryable, and renderable —
adopted directly from anvil-001 field feedback ("created from my pain but
never made available properly").

## What Ships

| Area | Detail |
| ---- | ------ |
| Lint | **W022** — `Packages:` entries that resolve to no workspace directory warn; silent off-monorepo (PKG-002, #106) |
| Next | `aps next --package <name>` filter + `--by-package` grouped queue with `(untagged)` bucket; tag inheritance, case-insensitive, path-qualified (PKG-001, #107) |
| Rollup | `aps rollup --by-package` — generated "Modules by Package" view (PKG-003, #108) |

All parity-bound: W022 in all three CLIs; next/rollup in Rust + bash (no
PowerShell surface). Shared fixtures wired into `test/cli-parity.sh`.

## Success Criteria

- [ ] `cargo publish --dry-run` from the release commit; all suites green
- [ ] Windows cross-check green (the v0.5.0 tag-failure guard, in CI since #105)
- [ ] Tag published; five targets + `SHA256SUMS`; crates.io at 0.6.0
- [ ] Scoop updated post-tag from `SHA256SUMS` (leave manifest at 0.5.0 during prep)
- [ ] CHANGELOG `[Unreleased]` promoted to `[0.6.0]`; release notes drafted
- [ ] Post-release sweep: PKG items + module `Merged → Complete` with evidence

## Risks

| Risk | Impact | Mitigation |
| ---- | ------ | ---------- |
| W022 warns on projects with stale tags | Low | Warning-only; gate keeps single-package projects silent |
| crates.io publish is irreversible | Medium | Dry-run from the exact release commit before tagging |
| Version drift across channels (D-036) | Medium | One prep PR bumps every channel; verify with the v0.5.0 checklist |

## Out of Scope

- PKG-003 `--write` (fenced in-place index refresh) — deferred by design.
- REL-005 `aps release` subcommand — Ready but not in this cut.
- anvil-001 epoch cut — parked (user decision, 2026-07-16).

## Rollout

1. Prep PR (like #104): bump 0.5.0 → 0.6.0 across Cargo.toml/.lock, config,
   package.json, README, installers, scaffold stamps, docs; promote CHANGELOG;
   draft `docs/release-notes/v0.6.0.md`.
2. Merge → `cargo publish --dry-run` → tag `v0.6.0` → workflow builds/publishes.
3. `cargo publish`; apply notes; Scoop from `SHA256SUMS`; smoke (binary,
   hash, installer, crates.io).
4. Closeout: this doc → Shipped with evidence; status sweep.
