## Actions

### Action 1 — Scaffold `cli/` Cargo project

**Purpose**
Establish the Rust crate that will host the TUI wizard and (eventually)
native lint + orch commands.

**Produces**

- `cli/Cargo.toml` with:
  - `name = "aps-cli"`, edition 2024, MSRV note
  - `clap = { version = "4", features = ["derive"] }`
  - `eddacraft-tui = { git = "https://github.com/EddaCraft/eddacraft-tui" }`
    (path override for local dev — see Notes)
- `cli/src/main.rs` — clap-based entry point with `--version`
- `cli/.gitignore` — ignore `target/`
- Root `.gitignore` updated if needed

**Checkpoint**
`cli/` directory exists with valid Cargo manifest and source file.

**Validate**
`ls cli/Cargo.toml cli/src/main.rs`

**Wave** 1

### Action 2 — `cargo check` clean

**Purpose**

**Validate**

**Wave** 1

### Action 3 — Implement `aps --version`

**Purpose**
Hello-world equivalent that proves clap is wired.

**Produces**

- `cli/src/main.rs` defining `#[derive(Parser)]` with version pulled from
  `Cargo.toml`
- Subcommand stubs: `init`, `lint`, `next` returning "not yet implemented"
  (so the tree is visible from day one)

**Checkpoint**
`cargo run --manifest-path cli/Cargo.toml -- --version` prints expected
version.

**Validate**
`cargo run --manifest-path cli/Cargo.toml -- --version`

**Wave** 2

### Action 4 — Verify eddacraft-tui imports

**Purpose**

**Checkpoint**
`cargo build --manifest-path cli/Cargo.toml` exits 0 and the dependency

**Validate**

## Notes

- For local development the eddacraft-tui dependency may use
  `path = "../../EddaCraft/eddacraft-tui"`. The committed Cargo.toml should
  use the git form so CI / external contributors can build. A
  `cli/Cargo.toml.local` or `[patch.crates-io]` block is one option for
  developer ergonomics.
- The eddacraft-tui crate uses Rust edition 2024. `cli/` should match.
- `cargo` may not be on PATH in every environment — TUI-001 is gated on
  the developer running cargo locally; CI integration is TUI-006's
  responsibility.
