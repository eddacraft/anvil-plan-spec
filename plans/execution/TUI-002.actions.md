# TUI-002 Implementation Plan

**Goal:** Implement the core `aps init` wizard sections for profile, project shape, and AI tooling.
**Architecture:** Keep wizard decisions in a testable state machine separate from terminal rendering. The live TUI renders the current section with EddaCraft shell chrome and select widgets, while unit tests verify navigation, conditional monorepo behavior, and tool-specific configuration gates.
**Tech Stack:** Rust 2024, clap, ratatui/crossterm, eddacraft-tui.

---

## File Map

- `cli/Cargo.toml` — add direct terminal/runtime dependencies used by the APS CLI.
- `cli/src/main.rs` — route `aps init` into the wizard runner.
- `cli/src/wizard.rs` — wizard state, section transitions, input handling, rendering, and tests.
- `plans/modules/tui.aps.md` — link this action plan and record implementation results after verification.
