# APS v0.6.0 — The Tagged Tier, For Real

v0.5 scaled APS to federated plan trees; v0.6 gives the *default* monorepo
tier — one canonical index with `Packages:` scope tags — the tooling it never
had. Tags are now validated, queryable, and renderable.

## Highlights

- **W022 — tag validation**: a `Packages:` entry that resolves to no workspace
  directory warns as a likely typo (checked as given, then under `packages/`
  and `apps/`). Silent in single-package projects. All three CLIs.
- **`aps next --package <name>`** — the next ready item within one package's
  scope, with tag inheritance from the module column; **`--by-package`**
  renders the whole ready queue grouped by package, untagged items in an
  explicit `(untagged)` bucket.
- **`aps rollup --by-package`** — the "Modules by Package" index view
  generated from live metadata, ready to paste; generated views cannot drift.

## Install / upgrade

```bash
cargo install aps-cli            # or: cargo binstall aps-cli
curl -fsSL https://raw.githubusercontent.com/eddacraft/anvil-plan-spec/main/scaffold/install | bash -s -- --cli
# Scoop (Windows)
scoop install https://raw.githubusercontent.com/eddacraft/anvil-plan-spec/main/packaging/scoop/aps.json
```
