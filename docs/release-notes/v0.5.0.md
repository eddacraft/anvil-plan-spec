# APS v0.5.0 ‚Äî Federated Plans & Full-Lifecycle Tooling

APS can now scale from one plan tree to a federation of independently owned
package plans. A parent can link child plans, lint and orchestrate the whole
tree, and roll up progress without taking away each child's ability to work
standalone. The native binary also gains the complete APS lifecycle, upgrades
become explicit and reversible, and Rust/bash/PowerShell parity is enforced
behaviorally in CI.

## Highlights

- **Federated nested plans** ‚Äî co-located child plans, path-qualified
  cross-tree references, transitive lint, deterministic collision warnings,
  and standalone child operation.
- **Cross-tree orchestration** ‚Äî `next`, `start`, `complete`, `graph`, and
  `audit` understand federations and `--child`; `rollup` summarizes each child
  from the parent.
- **Scaffold and guidance** ‚Äî create the layout with `aps init --scope nested`
  or `aps init --templates index-nested`; use the shipped templates, migration
  guide, and storefront example as the reference shape.
- **Full native lifecycle** ‚Äî Rust implementations of `start`, `complete`,
  `graph`, and `audit` close the binary-first gap left in v0.4.
- **Safer maintenance** ‚Äî `aps update` reconciles generated project files and
  reports every decision; `aps migrate` dry-runs and backs up before removing
  vendored CLI files.
- **Three-CLI parity** ‚Äî conductor and federated lint rules agree across Rust,
  bash, and PowerShell, with ordered findings compared over the shared fixture
  corpus in CI.
- **Canonical APS loop skills** ‚Äî installed Claude and Codex workflows cover
  planning, autonomy policy, safety, probe, resume, escalation, and landing.

## Install

```sh
# Install script (auto-detects platform)
curl -fsSL https://raw.githubusercontent.com/eddacraft/anvil-plan-spec/main/scaffold/install | bash

# Pin this version
VERSION=0.5.0 curl -fsSL https://raw.githubusercontent.com/eddacraft/anvil-plan-spec/main/scaffold/install | bash -s -- --cli

# Prebuilt binary, no build
cargo binstall aps-cli

# Build from source
cargo install aps-cli --version 0.5.0

# Scoop (Windows)
scoop install https://raw.githubusercontent.com/eddacraft/anvil-plan-spec/main/packaging/scoop/aps.json
```

Verify: `aps --version` ‚Üí `aps 0.5.0`.

## Upgrading from v0.4

- Run `aps update` to preview and reconcile the APS-owned project footprint.
- Run `aps migrate` first if the project still vendors `bin/` / `lib/`; add
  `--apply` only after reviewing the dry run.
- No plan-document migration is required. Existing single-tree and
  tagged-module plans remain valid; nested plans are opt-in.
