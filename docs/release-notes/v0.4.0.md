# APS v0.4.0 — Crosscutting Coordination & Binary-First Distribution

v0.3 turned APS into a workflow tool. **v0.4 makes it a coordinating one — and ships it like a real product.** The headline is the new **conductor module type** for concerns that span multiple modules; around it, APS becomes a global binary you install from crates.io, Scoop, or `cargo binstall`, gains release-planning and compound-engineering archives, and exposes its CLI to agents over MCP.

## Highlights

- **🎼 Conductor / crosscutting module type** — a first-class `Type: Conductor` for concerns that coordinate work *across* vertical modules (releases, audits, perf budgets, migrations). Ships a template, parser support, two new lint rules (**W002** unresolved cross-module references, **W006** index/Type consistency), an optional index section, and a full guide at `docs/conductor-modules.md`. `release-planning` is the canonical example.
- **📦 Binary-first distribution** — install `aps` as a global binary via **crates.io**, **GitHub release binaries**, **Scoop**, or **`cargo binstall`**. `aps init` now scaffolds project content only; `.aps/config.yml` pins the toolchain (`cli_version`) and carries runtime path defaults. `aps doctor` diagnoses drift; `aps setup` picks optional integrations; `aps migrate` cleans legacy footprints. Migration path off the vendored CLI included.
- **🚢 Release planning addon** — `plans/releases/` layout, a release template, R001–R004 lint rules, `aps init` scaffolding, and a workflow guide.
- **🖥️ Rust CLI + TUI wizard** — Ratatui `aps init` wizard, native `aps lint`/`aps next` at bash parity, and wizard hardening (bracketed paste, Windows key-release filter).
- **🤖 MCP server + conductor agent** — an optional `mcp/` package exposes the `aps` CLI surface to MCP-capable agents; ORCH-005 adds a coordinator agent role.
- **🧹 Self-hosting hygiene** — `aps audit` (runs Complete items' validations, flags A002/A003/A004), new hygiene lint rules (W017/W018/W019), a reconciled status vocabulary (SPEC-001), Claude Code Tasks mapping (TASKS-001), and AGENTS.md plan-update guidance (DOGFOOD-003).

## Install

```sh
# Install script (auto-detects platform)
curl -fsSL https://raw.githubusercontent.com/eddacraft/anvil-plan-spec/main/scaffold/install | bash

# Pin this version
VERSION=0.4.0 curl -fsSL https://raw.githubusercontent.com/eddacraft/anvil-plan-spec/main/scaffold/install | bash -s -- --cli

# Prebuilt binary, no build (from GitHub releases)
cargo binstall aps-cli

# Build from source (needs the Rust toolchain)
cargo install aps-cli

# Scoop (Windows)
scoop install https://raw.githubusercontent.com/eddacraft/anvil-plan-spec/main/packaging/scoop/aps.json
```

Verify: `aps --version` → `aps 0.4.0`.

## Upgrading from v0.3

- Existing projects move from the vendored CLI to the global binary (the install script's `--upgrade` path). Pin the toolchain per project with `cli_version: 0.4.0` in `.aps/config.yml`.
- Hand-editing plans remains fully supported; the CLI is opt-in.
